# brand-locked-ai-prototyping

## Trigger
Use when a project needs to apply the workflow in [[Claude Design as Brand-Locked AI Prototyping]] and capture evidence before turning it into a durable practice.

## Procedure
1. Define the smallest real task and success criteria.
2. Fill [[Brand-Locked Design System Prototype Kit/Brand Lock Trial Packet]] while running the task.
3. Attach concrete evidence: screenshots, logs, PRs, generated files, or demo links.
4. Decide promote / iterate / hold / retire.
5. Patch the package README only with claims backed by the captured evidence.

## Pitfalls
- Do not treat a polished demo as validation.
- Do not skip proof links.
- Do not promote this draft until at least one real trial succeeds.
